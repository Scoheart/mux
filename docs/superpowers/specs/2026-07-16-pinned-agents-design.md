# Pinned Agents Design

## 目标

为 MUX 顶栏增加常用 Agent 快捷入口。用户可以从完整 Agent 选择器中置顶最多 6 个 Agent、调整顺序，并通过一次点击进入对应 Agent 配置页。

该功能必须保留现有完整 Agent 搜索和自定义 Agent 入口，并在 `1200x820` 与最小 `900x600` 窗口中维持顶栏完整、稳定、无横向溢出。

## 已确认决策

- 使用“独立快捷栏 + 完整选择器”结构，不把快捷图标嵌入选择器，也不以图标菜单替代完整选择器。
- 顶栏最多置顶 6 个 Agent。
- Agent 列表行内提供 Pin 操作。
- 已置顶 Agent 支持拖拽排序。
- 点击顶栏图标直接进入对应 Agent 配置页；点击当前 Agent 不刷新页面。
- 完整选择器适当收窄，不再常驻显示 Agent ID。

## 顶栏结构

置顶栏位于完整 Agent 选择器左侧，二者共享顶栏中部区域，但保持独立交互边界。

### 置顶栏

- 最多显示 6 个品牌图标，顺序与用户保存顺序一致。
- 常规窗口中图标为 `30px`，窄窗口中为 `28px`；使用固定尺寸和间距，选中状态不得改变布局。
- 当前 Agent 使用细边框和底部短线表达选中，不增加常驻文字。
- 每个图标提供包含完整 Agent 名称的 tooltip 和可访问名称。
- 点击图标调用既有 Agent 导航，直接进入统一 Agent 配置页。
- 没有置顶项时不保留空白占位，完整选择器自然占据该区域。

### 完整选择器

- 关闭状态常规宽度为 `220px`，窄窗口为 `170px`。
- 仅显示 Agent 图标、单行名称和下拉箭头；Agent ID 在下拉内容中展示，不占用顶栏宽度。
- 未选择 Agent 时显示“选择 Agent”。
- 下拉面板宽度独立维持 `390px`，不跟随关闭状态控件缩窄。
- 面板继续提供搜索、完整可配置 Agent 列表和“添加自定义 Agent”入口。

### 响应式边界

- `1200x820` 显示 6 个常规尺寸图标和 `220px` 选择器。
- `900x600` 显示 6 个较小图标和 `170px` 选择器。
- 不隐藏 MCPs、Models、主题、刷新或检查更新操作，也不通过水平滚动容纳顶栏。
- 名称过长时在关闭状态单行截断，并通过 tooltip 提供完整名称。

## 下拉管理交互

未搜索时，下拉面板按以下顺序组成：

1. 搜索栏。
2. “已置顶”区域。
3. “全部 Agent”区域。
4. “添加自定义 Agent”入口。

### 已置顶区域

- 最多 6 行，显示图标、名称、Agent ID、拖拽手柄和取消置顶按钮。
- 支持拖拽排序；保存成功后顶栏立即按新顺序展示。
- 拖拽手柄可聚焦；聚焦时使用 `Option+ArrowUp` 和 `Option+ArrowDown` 调整顺序。
- 没有置顶项时显示一行简短引导，不展示空列表框。

### 全部 Agent 区域

- 默认排除已经出现在“已置顶”区域的 Agent，避免重复。
- 其余可配置 Agent 继续按名称排序。
- 行悬停或键盘聚焦时显示 Pin 图标。
- 达到 6 个后，其他 Agent 的 Pin 按钮禁用，并提供“最多置顶 6 个”的说明。

### 搜索状态

- 搜索时合并为一份匹配结果，不再分区，避免同一 Agent 重复出现。
- 每个结果仍显示置顶状态并允许置顶或取消置顶。
- 搜索状态下禁用拖拽排序；清空搜索后恢复“已置顶/全部 Agent”结构。

## 数据模型

置顶数据属于 MUX 自有 UI 偏好，保存在 `~/.mux/settings.json`：

```json
{
  "ui": {
    "pinned_agents": ["claude-code", "codex", "qoder", "pi"]
  }
}
```

新增强类型 `ui` section，并保留向后兼容的可选字段语义。旧 settings 没有该 section 时等价于空置顶列表。

`pinned_agents` 是有序 Agent ID 数组，必须满足：

- 长度不超过 6。
- ID 不重复。
- ID 对应当前可配置的 Agent；“可配置”沿用现有 catalog 定义，即 `has_global = true`。
- 顺序即顶栏与“已置顶”区域的展示顺序。

不使用 `localStorage` 作为 source of truth，避免 WebView 缓存清理后丢失偏好，也避免 React 绕过现有 settings 安全写入边界。

## 组件与命令边界

### Rust core

- 定义 UI settings 数据结构及最大置顶数量常量。
- 提供读取和提交有序置顶列表的操作。
- 校验数量、唯一性和 Agent 可配置性。
- 读取时忽略已不再可配置的失效 ID；下一次成功提交时只保存当前有效列表，从而规范化清理失效项。
- 继续使用现有 settings 锁、严格解析、未知字段透传、私有权限、原子替换和乐观并发检查。

### Tauri command

- 暴露读取置顶列表和提交有序列表的薄命令。
- 不复制校验或持久化逻辑。
- 只返回 Agent ID，不返回或读取 Agent 配置内容。

### React

- `Layout` 负责呈现顶栏快捷栏和调用既有 Agent 导航。
- Agent picker 负责搜索、置顶操作和排序交互。
- 置顶状态通过独立 hook/API 从 core 加载和更新，不由组件直接访问文件或 `localStorage`。
- 排序中的临时视觉状态可以留在 React；只有 core 保存成功后才确认新顺序。

如果现有 `Layout` 中 picker 代码继续增长，实施时应将下拉内容提取为聚焦组件，但不做与置顶功能无关的顶栏重构。

## 数据流

### 启动

1. 桌面端加载可配置 Agent 列表。
2. 同时读取保存的有序置顶 ID。
3. React 将有效 ID 映射为当前 Agent 数据，并渲染快捷栏。

### 置顶或取消置顶

1. 用户点击 Agent 行右侧 Pin。
2. React 生成候选有序 ID 列表并提交给 Tauri。
3. core 校验并原子写入 `settings.ui.pinned_agents`。
4. 成功后 UI 采用新列表；失败则保留原状态并提示原因。

### 排序

1. 用户在“已置顶”区域拖拽或使用键盘调整顺序。
2. React 展示临时顺序并提交完整有序 ID 列表。
3. 保存成功后顶栏同步；失败时恢复上一次已保存顺序。

## 错误处理

- 数量超过 6：core 拒绝写入；UI 在发起前禁用第 7 个 Pin，并保留 core 防线。
- 重复、未知或 `has_global = false` 的 Agent ID：core 拒绝写入并返回明确错误。
- settings 文件损坏：沿用严格解析，拒绝覆盖原文件。
- 并发修改或写入失败：不确认候选状态，恢复最后一次成功列表并显示 toast。
- 品牌图标缺失：沿用 `AgentGlyph` fallback；置顶功能不引入新的图标 source of truth。
- Agent 数据刷新后发现失效 ID：显示层忽略失效项，读取流程不静默改写 settings；下一次用户主动保存置顶列表时提交规范化后的有效 ID。

## 可访问性

- 顶栏图标按钮有 Agent 名称、选中状态和 tooltip。
- Pin、取消置顶和拖拽手柄均可通过键盘聚焦。
- 排序必须提供键盘等价操作，并在成功后通过可访问状态文本反馈新位置。
- 下拉搜索、列表导航、Escape 关闭和点击外部关闭继续沿用现有行为。
- 激活与禁用状态不能只依赖颜色表达。

## 测试与验收

### Core 测试

- 空 section 与旧 settings 兼容。
- 1–6 个有效 ID 保持输入顺序并可往返。
- 第 7 个、重复、未知和不可配置 ID 被拒绝。
- 修改 `ui.pinned_agents` 保留 settings 其他 section 和未知字段。
- 读取时忽略失效 ID，下一次有效变更会清理失效记录。
- 损坏 settings 和并发变化仍然拒绝覆盖。

### UI 测试

- Pin/取消 Pin 构造正确顺序。
- 排序成功同步顶栏，失败恢复原顺序。
- 达到上限后普通 Pin 禁用，取消置顶仍可用。
- 搜索模式不重复 Agent，并禁用排序。
- 当前 Agent、长名称和无置顶状态正确呈现。

### 真实应用验收

- `1200x820` 和 `900x600` 下 6 个置顶图标、收窄选择器和所有顶栏操作均可见，无横向溢出或文字重叠。
- 浅色与深色主题中边界、选中态和 tooltip 清晰。
- 点击每个置顶图标进入正确 Agent 配置页，不刷新当前 Agent。
- 下拉框搜索、Pin、取消 Pin、拖拽、键盘排序和上限提示可用。
- 重启正式安装版后顺序保持。
- 操作置顶不会修改任何 Agent MCP/model 配置或 Keychain 数据。

## 非目标

- 不把置顶 Agent 同步到云端或其他电脑。
- 不改变 Agent catalog、发现顺序或可写能力。
- 不在 CLI/TUI 中增加置顶导航界面。
- 不自动根据最近使用频率修改置顶列表。
- 不允许超过 6 个后使用 `+N` 顶栏溢出菜单。
