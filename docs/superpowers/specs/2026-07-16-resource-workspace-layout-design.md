# Resource Workspace Layout Design

## 目标

优化 MCPs 与 Models 的共享资源工作区，降低筛选和查看详情时的布局跳动：

- 状态筛选位于资源列表上方，不占用左侧导航空间。
- 左侧导航可调整宽度、不可关闭，并在重启后恢复用户宽度。
- 详情以覆盖式抽屉呈现，不压缩资源网格。
- MCPs 与 Models 复用一致的交互结构。
- 详情预览不得显示敏感配置值。

## 信息架构

共享工作区保持三层结构：

1. 左侧导航负责资源分类，不承担状态筛选。
2. 主区域顶部负责搜索、状态筛选和资源命令。
3. 主区域内容负责卡片网格与覆盖式详情抽屉。

MCPs 左侧导航只显示来源及来源管理操作。Models 左侧导航只显示协议。两类资源的状态筛选均进入列表上方的紧凑 Tabs。

## 状态 Tabs

`ResourceWorkspace` 增加独立的 `filters` 插槽，位于搜索工具栏下方、滚动内容上方。它不随卡片列表滚动。

MCPs 显示：

- 全部
- 使用中
- 未使用
- 被覆盖；数量为零时仍保留，避免筛选位置变化

Models 显示：

- 全部
- 已分配
- 未分配

每个 Tab 显示当前分类范围内的数量。MCP 来源改变后，Tabs 数量按当前来源重新计算。切换搜索、来源、协议或状态时关闭已打开详情，避免详情与当前列表范围不一致。

Tabs 使用下划线或轻量选中背景表达状态，不做大面积胶囊按钮。支持点击、键盘焦点及 `aria-selected`。

## 可调整左侧栏

共享工作区在左侧栏右边缘提供拖动热区和可见但克制的分隔反馈：

- 默认宽度：`224px`
- 最小宽度：`184px`
- 最大宽度：`340px`
- 不提供关闭或折叠入口
- 拖动期间禁用文本选择，并使用列调整光标
- 宽度保存到 `localStorage` 的 `mux.resourceWorkspace.sidebarWidth`；MCPs 与 Models 共用一个宽度，切换产品区时不发生跳动

读取到无效或超范围值时回退到默认值。窗口缩小时，实际宽度需要受可用空间约束，保证主区域不发生横向溢出。

## 覆盖式详情抽屉

详情抽屉从主内容区右侧覆盖卡片，不再作为 flex/grid 子项参与宽度计算。打开和关闭前后，卡片列数、卡片宽度和滚动位置保持不变。

- 宽度：`clamp(400px, 38vw, 520px)`
- 小窗口宽度：`min(520px, calc(100% - 32px))`
- 使用不透明背景、左边框和轻阴影
- 仅资源内容区显示轻遮罩，点击遮罩关闭
- `Escape` 关闭抽屉；存在更上层 dialog 时只关闭 dialog
- 抽屉头部和底部操作固定，正文独立滚动
- 打开另一张卡片时直接替换抽屉内容，不改变抽屉尺寸

遮罩阻止对底层卡片的误操作，但不影响应用顶栏。抽屉必须位于 workspace stacking context 内，不覆盖全局 Agent picker、更新提示或顶层 dialog。

## 敏感配置

配置预览在进入 React 视图前生成脱敏副本。字段名不区分大小写匹配 `token`、`secret`、`password`、`api_key`、`access_key`、`private_key`、`credential` 时，值显示为统一掩码，不把原值写入 DOM、title、日志或剪贴板预览。

现有“复制配置”命令维持原有行为边界，不从脱敏后的 DOM 反向组装数据。若产品后续需要复制含凭据配置，应单独设计确认和权限流程，本次不新增凭据展示入口。

## 组件边界

- `ResourceWorkspace`：管理共享布局、筛选插槽、侧栏宽度和详情 overlay 容器。
- `WorkspaceSidebar`：呈现导航内容及 resize handle，不持有业务筛选状态。
- `ResourceTabs`：共享的状态 Tabs 组件，接收选项、当前值和变更回调。
- `ResourceInspector`：保持详情内容 API，外层改为遮罩与绝对定位抽屉。
- `RegistryView`：维护来源、状态和 MCP 详情状态，计算来源范围内计数。
- `ModelsView`：维护协议、状态和模型详情状态。
- 配置脱敏 helper：纯函数，递归处理对象和数组，并由单元测试覆盖。

业务状态仍由 MCPs/Models 页面持有；共享组件只负责布局和无业务含义的 UI 状态。

## 验收标准

- 在 `1200x820` 和 `900x600` 下，打开详情前后资源网格列数不变化、无横向滚动。
- 状态筛选不再出现在左侧栏；MCPs 左栏只显示来源，Models 左栏只显示协议。
- 左侧栏可拖动到边界值，重启应用后恢复，且不能关闭。
- 点击遮罩和按 `Escape` 均可关闭详情；打开 dialog 时 `Escape` 不同时关闭详情。
- 搜索或切换筛选后，不残留已离开当前范围的详情。
- 详情 DOM 和截图中不包含敏感配置原值。
- 浅色与深色主题均保持不透明、清晰的边界和文本对比度。

- 运行 desktop 构建及现有测试。
