# Skills 工作区视觉简化设计

## 目标

减少 Skills 工作区中过量的边框和分隔线，让导航、工具区与资源卡片依靠背景、留白和排版形成层级。保留现有信息、交互与响应式能力，不改变 Skills 生命周期和后端行为。

## 视觉方向

采用“柔和分区 + 扁平卡片”。界面继续使用 MUX 现有的不透明中性色体系，不引入渐变、透明玻璃或装饰性线条。

- 侧栏以轻微背景差区分导航区域，移除标题下方横线；侧栏与内容区不再使用常驻竖向分隔线。拖动调整宽度时仍显示临时蓝色反馈线。
- 搜索与主操作、状态筛选位于同一个连续工具区域。两行之间不设横线，工具区与资源内容通过背景和间距过渡。
- 卡片移除左侧风险色条、内部来源框和 footer 横线。卡片边界改为低对比度表面与轻微阴影；hover、focus 和 selected 状态仍提供明确反馈。
- 卡片信息保持三层：名称与图标；来源与 revision；风险、库存状态与受影响 Agent。风险只通过紧凑标签表达，不再影响卡片轮廓。

## 范围

本轮只调整 Skills 工作区及 Skills 卡片的视觉表现。共享 `ResourceWorkspace` 的结构和交互不变；如果减少共享边框会影响 MCPs 或 Models，则使用 Skills 作用域样式覆盖，避免无意改变其他产品区。

不修改文案、数据模型、筛选逻辑、详情面板、安装/审阅对话框或 core 行为。

## 可访问性与响应式

- 卡片继续支持 Enter、Space、`aria-pressed` 与可见的键盘焦点。
- 状态不能只靠颜色表达，现有文字标签保留。
- `1200x820` 与 `900x600` 下不得出现横向溢出、顶部操作裁切或卡片内容重叠。
- `prefers-reduced-motion` 下不新增动画依赖。

## 验证

- 添加或更新组件测试，锁定 Skills 工作区的作用域类名和卡片语义结构，防止样式误扩散到 MCPs / Models。
- 运行 Desktop 测试与生产构建。
- 按 MUX UI review 规则，仅在最新稳定 `/Applications/MUX.app` 上检查 `1200x820`、`900x600`、横向 overflow 和实际截图；不得使用 Preview、dev server 或合成 IPC 页面替代正式安装版。
