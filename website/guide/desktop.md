# 桌面 App 图文教程

桌面 App 是 MUX 的可视化前端（macOS，Tauri + React），用鼠标浏览目录、勾选安装、编辑条目。它和命令行共享 `~/.mux/`，两端实时同步。

> 还没装？看 [安装](/guide/install#桌面-app-macos)。

## 界面总览

打开 App 默认进入 **Registry（目录）** 页。整体分三块：顶部导航、左侧来源栏、中间目录网格。

![MUX 界面总览](/img/registry-overview.png)

| # | 区域 | 说明 |
|:-:|---|---|
| **1** | **Registry 标签** | 当前页面。点顶部导航在 Registry / 各 Agent 页之间切换。 |
| **2** | **Agent 图标条** | 所有支持的 agent（Claude Code、Cursor、Codex…）。点一个进它的页面，管理它装了哪些 MCP。图标条可横向滚动。 |
| **3** | **主题 / 刷新 / 新增 Agent / 版本** | 月亮=深浅色切换；⟳=重新扫描各 agent 配置；➕=新增自定义 agent。 |
| **4** | **全部 / 生效中** | 目录的两个总入口，见下方[「全部 vs 生效中」](#全部-vs-生效中)。 |
| **5** | **各来源** | 你订阅/导入/手动/探索出来的来源，点任意一个只看它的条目。 |
| **6** | **搜索** | 按名字 / 描述过滤当前列表。 |
| **7** | **粘贴配置 / 新建 MCP** | 粘一段 `mcpServers` 自动识别；或手动新建一个条目。 |
| **8** | **MCP 卡片** | 目录里的一个 MCP，见下方[「卡片怎么看」](#卡片怎么看)。 |

## 全部 vs 生效中

目录是你所有来源的**并集**。同一个 server 可能同时存在于多个来源（比如「官方精选合集」和你「手动添加」里都有 `context7`）——这时按优先级 **手动 > 探索 > 订阅/本地** 只有一个「生效」。左侧两个入口就是为此设计的：

- **全部** —— 列出**每个来源里的每一份副本**，一个都不省略。被更高优先级覆盖的那份会标上 **`已被覆盖`**。
- **生效中** —— 只列出去重后真正生效的那一份。

点某个具体来源（如「官方精选合集」），则显示**该来源的全部条目**（含被覆盖的），方便你核对这个来源到底提供了什么。

::: tip
来源没有「开关」——目录只负责展示每个来源有哪些 server。要增减内容，用「添加 / 删除」或管理对应来源，而不是启停。
:::

## 卡片怎么看

![MCP 卡片结构](/img/registry-card.png)

| # | 部件 | 含义 |
|:-:|---|---|
| **1** | 头像 | 由名字生成的彩色图标，方便快速辨识。 |
| **2** | 名称 | MCP 的名字（`name`）。 |
| **3** | 传输 + 来源 | `STDIO`/`HTTP` 传输标签，加来源徽章（订阅 / 本地 / 手动添加 / 来自某 agent）。 |
| **4** | 端点 | stdio 的启动命令，或 http 的 URL（单行截断，悬停看全）。 |
| **5** | 使用状态 | 绿点=已被 N 个 agent 使用；灰点=未使用。 |

把鼠标移到卡片上，右下角会浮现**复制配置 / 编辑 / 删除**按钮（编辑、删除只对「手动添加 / 自动探索」的条目可用——订阅/本地来的是只读的）。点卡片本身打开详情。

## 安装到 agent

1. 顶部 **Agent 图标条**（#2）点一个 agent，进它的页面。
2. 点右上角 **添加 MCP**，在弹出的列表里搜索、选一个 MCP。
3. 确认写入。MUX 会：
   - **先备份**该 agent 的配置文件到 `~/.mux/backups/`；
   - 只**改写这一个 server**，保留同文件里其它 server 的原始字节；
   - 用该 agent 各自的格式（JSON / TOML）和键名写入。

## 开 / 关 / 删除

在某个 agent 的页面里，每个已装 MCP 都有一个开关和删除按钮：

- **停用**：从 agent 文件移除，但**记住配置**——随时开回来。
- **启用**：把停用的写回去。
- **删除**：从该 agent 彻底卸载。

在目录里对**手动 / 探索**条目，还有「删除」把它从目录彻底移除并从所有 agent 卸载。

## 编辑与粘贴

- **编辑**：可视化改一个 MCP 的字段。保存后自动**传播**到「干净」安装了它的 agent（手改过的、项目级的不动）。
- **粘贴**（顶部 #7「粘贴配置」）：直接粘一段 `{"mcpServers": {…}}`（或 TOML），MUX 识别其中的 server 并加入「手动添加」。
- **重新同步**：编辑器里的按钮，把当前配置显式推给所有已安装该 MCP 的 agent；被定制过的会跳过并报告，可选强制覆盖。

## 来源管理

左侧来源栏顶部四个按钮（#5 上方）：**官方精选合集 / 导入本地文件 / 导出手动 MCP / 订阅远程 URL**。把鼠标移到某个来源行上，会浮现**刷新 / 删除**。

- **订阅**：填一个指向 MCP 配置文件的 URL，其中的 server 加入目录。
- **导入**：选一个本地配置文件。
- **导出**：把你「手动添加」的所有 MCP 导出成一个配置文件，便于分享/备份。
- **官方精选合集**：一键订阅内置的整理清单。

## Agent 管理

- **新增自定义 agent**（顶部 ➕，#3）：起名、指定配置文件路径、选格式和配置键名。
- **编辑** agent 的配置路径（路径以 `~/…` 可移植方式存储）。

内置的 21 个 agent 见 [支持的 Agent](/guide/agents)。

## 安全性

- **原子写入** + 每次触碰 agent 配置前的**时间戳备份**（`~/.mux/backups/`）。
- **按条目改写**，绝不整体重写 `mcpServers`，保护同文件里的其它 server。

命令行做同样的事，只是在终端里 → [命令行 / TUI](/guide/cli)。
